# 既知の落とし穴 (pitfalls)

挙動が想定と違う・自動化が変な失敗をする・調査して原因が分かった、といったときに残す場所。常時参照する必要はないが、同じ問題に再びぶつかったときに「ここを見れば答えがある」状態にする。

## 書き方

各ノートは **症状 / 原因 / 回避策 / 一次ソース** の 4 ブロックを基本構成にする。一次ソース URL は必ず添える（後で「これ本当？」となったときに検証できるように）。
